# Guides

Task-oriented how-tos. If the answer to "how do I...?" is not obvious from the code, it belongs here.

**Nothing lives here yet.** Guides will be added as the project matures. Anticipated entries:

- `dev-setup.md` — install, run locally, run tests, build.
- `pack-authoring.md` — how to create a template pack or data pack by hand.
- `pack-tooling.md` — how to use `codex-pack` CLI (Phase 4a).
- `release-process.md` — version bump, changelog, deploy.
- `contributing.md` — how to propose changes, PR etiquette. (Will be promoted to a root-level `CONTRIBUTING.md` once there are external contributors.)

When writing a new guide:

1. Link it from this index.
2. Start with a single-line "what you'll get out of this" summary.
3. Assume the reader has cloned the repo and knows nothing else.
4. Prefer numbered steps; prose is for rationale, not instructions.
5. Include expected output for each command a reader will run.
